# Архитектура и концепция контекстной системы изучения слов

Данный документ описывает смену парадигмы в обучении иностранным языкам: переход от классического «словаря слов» (прямой перевод лексемы) к динамической системе многозначных слов в контекстах (облегченная версия Word Sense Disambiguation + Anki + корпус примеров).

---

## 🧠 1. Главное изменение парадигмы

Система больше **НЕ** работает со «словом как изолированной единицей».

Взаимодействие происходит на стыке трёх сущностей:

> **Word (Лексема)** + **Meaning (Значение)** + **Context Instance (Контекстный пример)**

---

## 🧩 2. Базовая модель данных

### 📦 Word (Лексема)

Представляет собой саму текстовую форму слова.

```json
Word {
  "id": "uuid",
  "text": "run"
}
```

### 🧠 Sense (Значение / Смысл)

Ключевая сущность, разделяющая разные семантические значения одного и того же слова.

```json
Sense {
  "id": "uuid",
  "wordId": "uuid",
  "definition": "упрощенное описание значения / перевод"
}
```

**Пример для слова `run`:**

- `run` → «бежать»
- `run` → «управлять»
- `run` → «работа / показ (a run of a show)»

### 🎬 Context Instance (Контекстное проявление)

Конкретный пример употребления слова в определённом значении, привязанный к источнику.

```json
ContextInstance {
  "id": "uuid",
  "source": "Friends S01E01",
  "sentence": "He runs the company",
  "wordId": "uuid",
  "senseId": "uuid"
}
```

---

## 🔁 3. Целевой результат обучения

Основная метрика прогресса меняется:

| Было | Стало |
|------|-------|
| «Я знаю перевод этого слова» | «Я встречал это слово в N разных смыслах и контекстах» |

---

## 📊 4. Структура карточки слова (Word Card)

Карточка слова превращается в агрегатор смыслов и контекстов.

**Пример карточки слова: `run`**

- **Meanings (Значения):**
  1. бежать *(12 примеров в базе)*
  2. управлять *(5 примеров в базе)*
  3. показывать (a run of show) *(2 примера в базе)*

- **Seen in (Встречено в медиа):**
  - Friends S01E01
  - Friends S01E03
  - Movie X

---

## 🧭 5. Динамическое формирование значения

**Ключевая продуктовая гипотеза:** значение слова формируется динамически из контекста.

- ❌ **Неправильно:** Брать статичный перевод из академического словаря.
- ✅ **Правильно:** Демонстрировать, как именно слово используется в реальных, уже просмотренных пользователем ситуациях.

---

## 🔁 6. Связь с колодами интервального повторения

Роль колод (Decks) переосмысляется:

1. 🎬 **Episode deck (Колода эпизода):** Линейный набор ContextInstances, встреченных в конкретной серии/видео.
2. 📚 **Word view (Общий вид слова):** Сводная агрегация всех ContextInstances по всем колодам для глубокого изучения лексемы.

---

## ⚙️ 7. Автоматическое «обогащение слова»

Система реализует следующую логику при обработке нового контента:

```javascript
if (wordExists && newContext != knownSenses) {
  createNewSense(); // Добавление нового семантического значения к слову
}
```

## 🧩 8. UX / Пользовательский опыт

### При просмотре эпизода

Пользователь выделяет слово `run`. Система отображает:

- бежать *(уже известно пользователю)*
- управлять бизнесом *(новое для пользователя значение!)*
- 🔘 Кнопка: «Добавить новое значение в свой словарь»

### Внутри карточки слова

- 📍 Friends S01E01: *"He runs the company"* → [управлять]
- 📍 Friends S01E03: *"He runs fast"* → [бежать]

---

## ⚠️ 9. Ограничения для MVP (Защита от переусложнения)

Чтобы проект не превратился в сложную академическую NLP-систему, на этапе MVP действуют жёсткие ограничения:

- ❌ **Исключить из MVP:** Автоматическое определение смысла через AI, построение сложных графов семантических связей.
- ✅ **Включить в MVP:** Ручное или полуручное назначение Sense пользователем *(функцию подсказать смысл через LLM/AI можно внедрить позже)*.

---

## 🧭 10. Итоговая архитектура системы

Иерархическая структура связей:

```
Word (Слово)
  └── Sense (Значение)
        └── ContextInstance (Контекстный пример)
              ├── source (Источник)
              └── sentence (Предложение)
```
